## 概要

システムのアーキテクチャ設計書を格納するディレクトリです。

## ドキュメント

### diagram.md
システム構成図と各レイヤーの説明

## アーキテクチャ原則

1. **レイヤー分離**: プレゼンテーション層、ビジネスロジック層、データアクセス層を明確に分離
2. **疎結合**: コンポーネント間の依存関係を最小化
3. **再利用性**: 共通機能はユーティリティとして抽出
4. **テスタビリティ**: テスト可能な設計を優先
