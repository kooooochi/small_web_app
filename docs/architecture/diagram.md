# システムアーキテクチャ

## 全体構成

```
┌─────────────────────────────────────┐
│        Presentation Layer           │
│         (Components)                │
│  ┌──────────┐      ┌──────────┐   │
│  │  Button  │      │  Header  │   │
│  └──────────┘      └──────────┘   │
└─────────────────────────────────────┘
              │
              ▼
┌─────────────────────────────────────┐
│       Business Logic Layer          │
│          (Utils)                    │
│  ┌──────────┐    ┌──────────┐     │
│  │ Helpers  │    │Validators│     │
│  └──────────┘    └──────────┘     │
└─────────────────────────────────────┘
              │
              ▼
┌─────────────────────────────────────┐
│      Data Access Layer              │
│        (Services)                   │
│  ┌──────────────────────┐          │
│  │    API Service       │          │
│  └──────────────────────┘          │
└─────────────────────────────────────┘
              │
              ▼
┌─────────────────────────────────────┐
│      External APIs                  │
│   (REST API Endpoints)              │
└─────────────────────────────────────┘
```

## レイヤー説明

### Presentation Layer（プレゼンテーション層）
- UIコンポーネント
- ユーザーインタラクション処理
- 表示ロジック

### Business Logic Layer（ビジネスロジック層）
- データ変換・加工
- バリデーション
- 共通ユーティリティ

### Data Access Layer（データアクセス層）
- API通信
- データの取得・更新
- エラーハンドリング

## データフロー

1. ユーザーがUIコンポーネントを操作
2. コンポーネントがバリデーションを実行
3. API Serviceを通じて外部APIに通信
4. レスポンスデータをヘルパー関数で加工
5. UIコンポーネントに反映

## セキュリティ考慮事項

- API キーは環境変数で管理
- 入力値は必ずバリデーション
- XSS対策として出力エスケープ実施
- HTTPS通信を使用
