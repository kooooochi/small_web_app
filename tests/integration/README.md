# integration/

統合テスト

## 概要

複数のモジュールやコンポーネントの連携動作を検証する統合テストを格納するディレクトリです。

## テストファイル

### api.test.js
API Serviceの統合テスト例

## 統合テストの特徴

### ユニットテストとの違い

- **スコープ**: 複数のモジュールの連携をテスト
- **実行時間**: ユニットテストより時間がかかる
- **外部依存**: APIやデータベースなど外部システムとの連携をテスト

### テストの対象

1. API通信の完全なフロー
2. コンポーネント間のデータ受け渡し
3. エラーハンドリングの連携
4. 実際のユーザーシナリオ

## モックとスタブ

外部APIへの実際の通信を避けるため、モックサーバーを使用：

```javascript
// モックサーバーのセットアップ例
beforeAll(() => {
  mockServer.listen();
});

afterAll(() => {
  mockServer.close();
});
```

## 実行方法

```bash
# このディレクトリのテストのみ実行
npm run test:integration
```
